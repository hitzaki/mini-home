# 测试数据生成指南

## 快速生成测试数据

### 步骤1：生成加密密码

运行 `PasswordGenerator.java` 生成以下测试用户的密码：

1. **admin用户** - 密码：`123456`
2. **testuser用户** - 密码：`123456`

或者使用命令行：
```bash
# 生成admin用户的密码
java PasswordGenerator 123456

# 生成testuser用户的密码（相同密码，但会生成不同的加密结果）
java PasswordGenerator 123456
```

### 步骤2：更新test-data.sql

将生成的加密密码替换到 `test-data.sql` 文件中对应的位置。

### 步骤3：执行SQL

在数据库中执行 `test-data.sql` 文件。

## 测试数据说明

### 用户数据
- **admin** - 管理员账户，包含完整的应用和设置
- **testuser** - 测试用户，包含部分应用

### 应用数据
每个用户包含：
- 8-10个常用网站应用（GitHub、Google、百度、B站等）
- 部分应用在桌面显示，部分仅在应用列表中
- 不同的打开方式（内部窗口/外部浏览器）

### 用户设置
- PC端和移动端的不同设置
- 不同的主题配置

### 小组件
- 时钟组件
- 日历组件

### 回收站
- 一个已删除的应用示例

## 注意事项

1. 如果数据库已有数据，请调整SQL中的用户ID
2. 应用数据中的 `user_id` 需要与实际插入的用户ID对应
3. 密码必须使用BCrypt加密，不能直接使用明文
